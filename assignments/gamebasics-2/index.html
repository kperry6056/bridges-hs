Description

In this second game basics project, students use the BRIDGES library class called NonBlockingGame to create a player that can be moved around the game board with the upArrow, downArrow, rightArrow,and leftArrow keys.

Topics

  • 2D arrays
  • Using the Bridges API (NonBlockingGame class, NamedSymbol enum, NamedColor enum)
  • Event-Driven Programming

Intended learning outcomes

By the end of the two Game Basics projects, students will be able to…

  • Apply: Create objects by calling constructors with parameters. (AP CS A Unit 2, MOD-1.D)
  • Apply: Use typed variables in order to represent reference data. (AP CS A Unit 2, VAR-1.D)
  • Apply: Represent branching logical processes by using conditional statements. (AP CS A Unit 3, CON-2.C )
  • Apply: Use understanding of row and column while creating algorithms for nested iteration. (AP CS A Unit 4, CON-2.C, CON-2.E, CON-2.G )
  • Apply: Use understanding of “out of bounds” errors when debugging code. (AP CS A Unit 8, CON-2.N )
  • Apply: Use an API as reference in order to figure out how to use Java library classes.
  • Apply: Use an API and class headers as reference when determining inheritance relationships. (AP CS A Unit 9, MOD-3.B )
  • Analyze: Analyze library class code, applying understanding of abstract vs concrete classes, abstract vs concrete methods, constructors, and instance variables. (AP CS A Units 2, 5, & 9 )
